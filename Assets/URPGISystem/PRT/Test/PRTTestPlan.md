# PRT模块测试方案

## 测试目的
验证PRT（Precomputed Radiance Transfer）模块的功能是否正常工作，包括探针生成、数据捕获、实时渲染等核心功能。

## 测试环境
- Unity版本：2021.3.16
- 渲染管线：URP（Universal Render Pipeline）
- 平台：PC

## 测试步骤

### 1. 准备工作
1. 在Unity编辑器中打开项目
2. 确保URP管线已正确配置
3. 在场景中创建一个空GameObject，重命名为"PRTTest"
4. 给该GameObject添加"PRTTest"组件

### 2. 场景创建测试
1. 点击"PRTTest"组件的"Start PRT Test"上下文菜单
2. 验证测试场景是否成功创建，包括：
   - 地面、墙壁和天花板
   - 方向光
   - 测试物体（球体）
   - 探针体和探针网格

### 3. 探针生成测试
1. 检查场景中是否生成了探针网格
2. 验证探针数量是否符合配置（默认5x3x5=75个）
3. 检查探针的位置分布是否均匀

### 4. 数据捕获测试
1. 点击"PRTTest"组件的"Capture Probe Data"上下文菜单
2. 等待数据捕获完成（可能需要几秒钟）
3. 验证ProbeVolumeData.asset文件是否生成并包含数据

### 5. PRT效果验证
1. 点击"PRTTest"组件的"Verify PRT Effect"上下文菜单
2. 检查控制台是否输出验证成功信息
3. 确认探针数据大小是否合理

### 6. 实时渲染测试
1. 进入Play模式
2. 观察场景中物体的光照效果
3. 验证间接光照是否正确计算
4. 移动相机，观察光照效果是否随视角变化

### 7. 性能测试
1. 在不同探针密度下测试性能：
   - 低密度：3x2x3=18个探针
   - 中密度：5x3x5=75个探针
   - 高密度：8x4x8=256个探针
2. 记录不同密度下的帧率
3. 分析性能瓶颈

## 测试用例

### 测试用例1：基本功能测试
- **测试目标**：验证PRT模块的基本功能
- **测试步骤**：
  1. 执行场景创建测试
  2. 执行探针生成测试
  3. 执行数据捕获测试
  4. 执行PRT效果验证
- **预期结果**：所有测试步骤成功完成，无错误信息

### 测试用例2：实时渲染效果测试
- **测试目标**：验证PRT在实时渲染中的效果
- **测试步骤**：
  1. 完成基本功能测试
  2. 进入Play模式
  3. 观察场景光照效果
  4. 移动相机和测试物体
- **预期结果**：场景中的物体能够正确接收间接光照，光照效果自然

### 测试用例3：性能测试
- **测试目标**：验证PRT模块的性能表现
- **测试步骤**：
  1. 分别使用低、中、高密度探针网格
  2. 记录每种密度下的帧率
  3. 分析性能数据
- **预期结果**：在中密度探针网格下，帧率应保持在30fps以上

### 测试用例4：场景变化适应性测试
- **测试目标**：验证PRT对场景变化的适应能力
- **测试步骤**：
  1. 完成基本功能测试
  2. 在场景中添加新的物体
  3. 重新捕获探针数据
  4. 观察光照效果
- **预期结果**：新添加的物体能够正确影响周围的间接光照

## 故障排除

### 常见问题及解决方案

1. **探针未生成**
   - 检查Probe预制体是否正确创建
   - 检查ProbeVolume组件参数是否合理

2. **数据捕获失败**
   - 确保场景中有足够的光照
   - 检查计算着色器是否正确加载
   - 验证RenderTexture是否正确创建

3. **光照效果异常**
   - 检查PRTComposite RenderFeature是否正确添加到URP渲染器
   - 验证Composite材质是否正确配置
   - 尝试重新捕获探针数据

4. **性能问题**
   - 减少探针数量
   - 降低探针Cubemap的分辨率
   - 优化场景复杂度
