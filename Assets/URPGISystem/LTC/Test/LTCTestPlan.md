# LTC 算法测试方案

## 测试目标
验证 LTC (Linearly Transformed Cosines) 算法在 URP 管线中的实现正确性，确保区域光渲染功能正常工作。

## 测试环境
- Unity 版本：2021.3.16
- 渲染管线：Universal Render Pipeline (URP)
- 渲染模式：延迟渲染
- 平台：PC

## 测试内容

### 1. 功能测试

#### 1.1 基础功能测试
- **测试点**：验证 LTCAreaLight 组件能够正确添加到场景中
- **测试方法**：在场景中创建一个平面，添加 LTCAreaLight 组件，检查是否能正常工作
- **预期结果**：组件能正常添加，无报错

#### 1.2 光照效果测试
- **测试点**：验证区域光能够正确照亮场景
- **测试方法**：创建一个包含多个物体的场景，添加区域光，观察光照效果
- **预期结果**：物体表面能够被区域光正确照亮，呈现出柔和的光照效果

#### 1.3 阴影测试
- **测试点**：验证区域光的阴影功能
- **测试方法**：启用区域光的阴影功能，观察物体是否产生正确的阴影
- **预期结果**：物体能够产生柔和的区域光阴影

#### 1.4 多光源测试
- **测试点**：验证多个区域光同时存在时的效果
- **测试方法**：在场景中添加多个区域光，观察它们的光照效果
- **预期结果**：多个区域光能够正确叠加，无冲突

### 2. 性能测试

#### 2.1 帧率测试
- **测试点**：验证区域光对帧率的影响
- **测试方法**：在场景中添加不同数量的区域光，测量帧率变化
- **预期结果**：添加少量区域光时帧率变化不明显

#### 2.2 内存测试
- **测试点**：验证区域光对内存的使用情况
- **测试方法**：监控添加区域光前后的内存使用情况
- **预期结果**：内存使用合理，无内存泄漏

### 3. 兼容性测试

#### 3.1 URP 版本兼容性
- **测试点**：验证在不同 URP 版本下的兼容性
- **测试方法**：在不同版本的 URP 中测试区域光功能
- **预期结果**：在 URP 12.0+ 版本中正常工作

#### 3.2 平台兼容性
- **测试点**：验证在不同平台上的兼容性
- **测试方法**：在 PC、Mac 等平台上测试区域光功能
- **预期结果**：在所有支持的平台上正常工作

## 测试用例

### 测试用例 1：基础区域光测试
- **场景**：创建一个简单的场景，包含一个平面和一个立方体
- **步骤**：
  1. 添加 LTCAreaLight 组件到场景中
  2. 调整区域光的位置和大小
  3. 观察光照效果
- **预期结果**：立方体和平面能够被区域光正确照亮

### 测试用例 2：阴影测试
- **场景**：创建一个包含平面和立方体的场景
- **步骤**：
  1. 添加 LTCAreaLight 组件
  2. 启用阴影功能
  3. 调整阴影参数
  4. 观察阴影效果
- **预期结果**：立方体在平面上产生柔和的阴影

### 测试用例 3：多光源测试
- **场景**：创建一个包含多个物体的场景
- **步骤**：
  1. 添加多个 LTCAreaLight 组件
  2. 调整每个光源的颜色和位置
  3. 观察光照效果
- **预期结果**：多个光源能够正确叠加，产生丰富的光照效果

## 测试工具

- Profiler

## 测试流程
1. 准备测试场景
2. 运行测试用例

4. 分析测试数据
5. 生成测试报告

## 注意事项
- 确保测试场景中没有其他光源干扰
- 测试时使用默认的 URP 配置
- 记录测试环境的硬件配置，以便比较性能结果
